# Umbrela Qrel Handling

## Supported Built-In Qrels

| Name | Track | Passage Collection |
|------|-------|--------------------|
| `dl19-passage` | TREC Deep Learning 2019 | MS MARCO v1 passages |
| `dl20-passage` | TREC Deep Learning 2020 | MS MARCO v1 passages |
| `dl21-passage` | TREC Deep Learning 2021 | MS MARCO v2 passages |
| `dl22-passage` | TREC Deep Learning 2022 | MS MARCO v2 passages |
| `dl23-passage` | TREC Deep Learning 2023 | MS MARCO v2 passages |
| `robust04` | Robust Track 2004 | TREC Disks 4&5 |
| `robust05` | Robust Track 2005 | AQUAINT |

## Relevance Labels (0–3)

| Label | Meaning |
|-------|---------|
| **0** | Passage has nothing to do with the query |
| **1** | Passage seems related to the query but does not answer it |
| **2** | Passage has some answer for the query, but may be unclear or hidden amongst extraneous information |
| **3** | Passage is dedicated to the query and contains the exact answer |

These labels match the TREC Deep Learning track graded relevance scale.

## How Evaluate Works

1. Load the named qrel (`--qrel dl19-passage`)
2. Load the retrieval result file (`--result-file run.trec`)
3. Identify "holes" — query-document pairs in the result file but not in the qrel
4. Judge each hole using the selected backend
5. Merge LLM judgments into the original qrel → write modified qrel
6. Compute nDCG@10 for both original and modified qrels
7. Generate a confusion matrix comparing LLM labels to human labels (where both exist)

## Output Artifacts

| Artifact | Location | Description |
|----------|----------|-------------|
| Modified qrel | `modified_qrels/` | TREC qrel file with LLM labels merged |
| Confusion matrix | `conf_matrix/` | PNG image comparing LLM vs human labels |

## Judge Categories (`--judge-cat`)

Controls which relevance categories to judge. Default: `0,1,2,3` (all).

```bash
# Judge all pairs
umbrela evaluate --judge-cat 0,1,2,3 ...

# Only judge pairs in categories 2 and 3 (relevant documents)
umbrela evaluate --judge-cat 2,3 ...
```

## Pyserini Dependency

The `evaluate` command requires pyserini for:
- Loading qrel data: `get_qrels(qrel_name)`
- Mapping query IDs to text: `get_query_mappings(qrel_name)`
- Fetching passage text for judgment prompts
- Computing nDCG@10 scores

Install with:
```bash
uv sync --extra pyserini
# or
pip install umbrela[pyserini]
```

MS MARCO v1 passages (dl19, dl20) use pyserini's `LuceneIndexReader`.
MS MARCO v2 passages (dl21–dl23) use direct file access.

## Few-Shot Examples

When `--few-shot-count N` is set (N > 0):
- Examples are sampled from the qrel dataset (one per category)
- Each example shows query, passage, reasoning, and final score
- The `bing` prompt type includes BRELA-style step-by-step reasoning
- Examples are generated by `generate_examples_prompt()` in `qrel_utils.py`
